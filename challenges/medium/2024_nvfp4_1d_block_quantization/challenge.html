Implement NVFP4 (NVIDIA FP4) activation quantization with two-level scaling. NVFP4 is the native 4-bit format on NVIDIA Blackwell GPUs for low-precision inference. A float32 activation matrix x of shape M × K is quantized per-token (per-row): each row (token) is divided into blocks of BLOCK_SIZE contiguous elements along K, where M is the token count and K is the feature (contraction) dimension of the GEMM. Per-token granularity handles the wide dynamic range across tokens — a delimiter or early attention token can have activations orders of magnitude larger than the rest, and per-tensor scaling would either clip outliers or zero out normal tokens. A per-tensor FP32 global_scale captures the overall magnitude (normalizing the data distribution into UE4M3 range), and per-block UE4M3 scales capture the local distribution within each block. This two-level scheme provides more precision than pure power-of-2 (MXFP4) scaling, at the cost of one additional FP32 scalar per tensor. Blackwell tensor cores natively consume packed FP4 data together with UE4M3 block scales and the FP32 global scale, performing fused dequantize-and-accumulate in a single hardware instruction. Dequantization: \(\hat{x} = \text{FP4}(q) \times \text{UE4M3}(s_{\text{block}}) \times s_{\text{global}}\)

Formats

  • FP4 E2M1: 1 sign, 2 exponent (bias=1), 1 mantissa. 16 values: \(\{0, \pm 0.5, \pm 1, \pm 1.5, \pm 2, \pm 3, \pm 4, \pm 6\}\)
  • UE4M3 block scale: unsigned FP8 E4M3. 4 exponent (bias=7), 3 mantissa. Fractional scale, range \(2^{-9}\) to 448. One per block along K.
  • FP32 global scale: per-tensor scalar that normalizes the data into UE4M3 range, \(s_{\text{global}} = \text{amax} / (448 \times 6)\).
  • Packing: two 4-bit FP4 values packed per uint8 along the K dimension. Consecutive elements along K are contiguous in memory, enabling coalesced GPU access.

FP4 E2M1 Encoding Table

0x0= 0.0   0x1= 0.5   0x2= 1.0   0x3= 1.5
0x4= 2.0   0x5= 3.0   0x6= 4.0   0x7= 6.0
0x8=-0.0   0x9=-0.5   0xA=-1.0   0xB=-1.5
0xC=-2.0   0xD=-3.0   0xE=-4.0   0xF=-6.0

Algorithm

global_amax = max(|x[i,j]|) over all i, j
s_global = global_amax / (448.0 * 6.0)    # FP32

For each token m in [0, M):
  For each block g in [0, ceil(K / BLOCK_SIZE)):
    k_start = g * BLOCK_SIZE
    k_end = min(k_start + BLOCK_SIZE, K)
    amax = max(|x[m, k]|) over k in [k_start, k_end)
    s_block = amax / (6.0 * s_global)       # raw block scale
    scales[m, g] = encode_UE4M3(s_block)    # uint8
    decoded = decode_UE4M3(scales[m, g])
    For each k in [k_start, k_end):
      nibble = nearest_FP4(x[m, k] / (decoded * s_global))
    Pack nibble pairs along K

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • FP4 nibbles packed in y; UE4M3 block scales in scales (uint8)
  • FP32 global_scale is a single scalar for the entire tensor
  • All computation must occur on the GPU
  • Each (token, block) pair is independent and may be computed in parallel

Example

Input: \(M=2\) tokens with \(K=8\) features, BLOCK_SIZE=4: \[ \begin{bmatrix} 6.0 & 3.0 & 1.0 & -2.0 & 0.5 & -0.5 & 0.0 & 1.0 \\ -3.0 & 0.0 & 1.5 & 4.0 & 6.0 & -6.0 & 3.0 & 2.0 \end{bmatrix} \]

global_amax = 6.0
\(s_{\text{global}} = 6.0 / 2688.0 \approx 0.002232\)

Block scales (raw then UE4M3):
Token 0, Block 0 (\(k=0..3\)): amax=6.0, \(s_{\text{block}}=448.0\), UE4M3=0x7E
Token 0, Block 1 (\(k=4..7\)): amax=1.0, \(s_{\text{block}}=74.67\), UE4M3=0x69
Token 1, Block 0 (\(k=0..3\)): amax=4.0, \(s_{\text{block}}=298.7\), UE4M3=0x79
Token 1, Block 1 (\(k=4..7\)): amax=6.0, \(s_{\text{block}}=448.0\), UE4M3=0x7E

Output: \(y\) (\(M=2, K/2=4\)) = packed FP4 nibbles
scales (\(M=2, G=2\)) = UE4M3 block scales
global_scale = 0.002232...

Constraints

  • K must be even
  • 1 ≤ M ≤ 8,192
  • 1 ≤ K ≤ 65,536
  • BLOCK_SIZE in {8, 16, 32}
  • Input x is float32; output y and scales are uint8
  • global_scale is float32
  • All tensors stored in row-major order
  • Performance is measured with M = 4,096, K = 4,096, BLOCK_SIZE = 16
