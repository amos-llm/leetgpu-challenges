Implement per-token (per-row) FP8 E4M3 quantization of a float32 matrix. This is the FP8 activation quantization scheme used in vLLM FP8 and SGLang FP8 for serving LLMs on NVIDIA H100 GPUs. Given a float32 matrix \(x\) of shape \(M \times K\), compute per-row scales and encode each element to the OCP FP8 E4M3 format.

FP8 E4M3 Format

  • Bit layout: \([S|E_3|E_2|E_1|E_0|M_2|M_1|M_0]\), bias = 7, max normal = 448
  • Normal: \((-1)^S \times 2^{E-7} \times 1.M\)
  • Subnormal (exp=0000): \((-1)^S \times 2^{-6} \times 0.M\)
  • NaN: encode as 0x7F. Overflow: clamp to ±448

Algorithm

For each row m in [0, M):
  max_abs_m = max(|x[m,k]|) over all k in [0, K)
  scale_m = max_abs_m / 448.0
  If max_abs_m == 0: scale_m = 1.0
  For each k in [0, K):
    y[m,k] = fp8_e4m3(x[m,k] / scale_m)
  scales[m] = scale_m

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • FP8 encodings in y (uint8); per-row scales in scales (float32)
  • All computation must occur on the GPU
  • Rows are independent and may be computed in parallel

Example

Input: Matrix \(x\) (\(M=2, K=4\)): \[ \begin{bmatrix} 240.0 & -120.0 & 0.0 & 60.0 \\ 1.0 & 2.0 & 4.0 & 8.0 \end{bmatrix} \]

Row 0: max_abs=240.0, scale=0.5357...
\(y[0,0] = \text{fp8_e4m3}(448.0) = \text{0x7E}\)
\(y[0,1] = \text{fp8_e4m3}(-224.0) = \text{0xF6}\)
\(y[0,2] = \text{fp8_e4m3}(0.0) = \text{0x00}\)
\(y[0,3] = \text{fp8_e4m3}(112.0) = \text{0x6E}\)

Row 1: max_abs=8.0, scale=0.017857...
\(y[1,0] = \text{fp8_e4m3}(56.0) = \text{0x66}\)
\(y[1,1] = \text{fp8_e4m3}(112.0) = \text{0x6E}\)
\(y[1,2] = \text{fp8_e4m3}(224.0) = \text{0x76}\)
\(y[1,3] = \text{fp8_e4m3}(448.0) = \text{0x7E}\)

Output: Matrix \(y\): \[ \begin{bmatrix} \text{0x7E} & \text{0xF6} & \text{0x00} & \text{0x6E} \\ \text{0x66} & \text{0x6E} & \text{0x76} & \text{0x7E} \end{bmatrix} \]
scales = \([0.5357..., 0.017857...]\)

Constraints

  • 1 ≤ M ≤ 8,192
  • 1 ≤ K ≤ 32,768
  • Input x is float32; output y is uint8; scales is float32
  • All tensors stored in row-major order
  • Performance is measured with M = 4,096, K = 4,096
