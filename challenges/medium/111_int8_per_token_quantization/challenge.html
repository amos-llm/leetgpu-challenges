Implement dynamic per-token (per-row) symmetric INT8 quantization of a float32 matrix. This is the activation quantization scheme used in production LLM serving systems like vLLM and TensorRT-LLM, where each token's hidden state is independently quantized before being fed into INT8 matrix multiplication kernels. Given a float32 matrix \(x\) of shape \(M \times K\), compute per-row scales and INT8 quantized values.

Algorithm

For each row m in [0, M):
  max_abs_m = max(|x[m,k]|) over all k in [0, K)
  scale_m = max_abs_m / 127.0
  If max_abs_m == 0: scale_m = 1.0
  For each k in [0, K):
    y[m,k] = clamp(round(x[m,k] / scale_m), -128, 127)
  scales[m] = scale_m  (stored as FP32)

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • INT8 results in y; per-row FP32 scales in scales
  • All computation must occur on the GPU
  • Rows are independent and may be computed in parallel
  • Use FP32 for intermediate computation to ensure accuracy

Example

Input: Matrix \(x\) (\(M=2, K=3\)): \[ \begin{bmatrix} 3.0 & -1.0 & 0.0 \\ 1.0 & 2.0 & 3.0 \end{bmatrix} \]

Row 0: max_abs=3.0, scale=0.02362
\(y[0] = [127, -42, 0]\)

Row 1: max_abs=3.0, scale=0.02362
\(y[1] = [42, 85, 127]\)

Output: Matrix \(y\): \[ \begin{bmatrix} 127 & -42 & 0 \\ 42 & 85 & 127 \end{bmatrix} \]
scales = \([0.02362, 0.02362]\) (FP32)

Constraints

  • 1 ≤ M ≤ 8,192
  • 1 ≤ K ≤ 32,768
  • Input x is float32; output y is int8; scales is FP32
  • All tensors stored in row-major order
  • Performance is measured with M = 4,096, K = 4,096
