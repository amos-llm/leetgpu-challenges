Implement a GPU program that applies the orthonormal Walsh–Hadamard transform to every row of a batched input matrix. Given an input tensor of shape B × N, where N is a power of two, compute the transform of each row independently and write the result to the output tensor. This kernel appears in modern LLM inference pipelines (e.g. QuaRot, QuIP#), where activations are rotated by a Hadamard matrix before low-bit quantization to spread out outlier values.

For a single row \(x \in \mathbb{R}^N\) the transform is \(y = \frac{1}{\sqrt{N}} H_N x\), where \(H_N\) is the Hadamard matrix defined recursively by \(H_1 = [1]\) and \[ H_{2n} = \begin{bmatrix} H_n & H_n \\ H_n & -H_n \end{bmatrix}. \] The fast Walsh–Hadamard transform computes \(y\) in \(O(N \log N)\) work using a butterfly schedule: for each stage \(h \in \{1, 2, 4, \ldots, N/2\}\) and each pair of positions \((j, j + h)\) within the row, replace \((x_j, x_{j+h})\) with \((x_j + x_{j+h}, x_j - x_{j+h})\). After all stages, divide the row by \(\sqrt{N}\) for the orthonormal scaling.

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • The final result must be stored in the output tensor
  • The kernel must be entirely GPU-resident—no host-side fallback
  • N is guaranteed to be a power of two

Example

Input:  B = 1, N = 4
        input = [1.0, 2.0, 3.0, 4.0]

Output: output = [5.0, -1.0, -2.0, 0.0]

Each transform stage operates in place on the current row: stage \(h = 1\) maps \([1, 2, 3, 4] \to [3, -1, 7, -1]\); stage \(h = 2\) maps \([3, -1, 7, -1] \to [10, -2, -4, 0]\). Dividing by \(\sqrt{4} = 2\) yields the final output above.

Constraints

  • 1 ≤ B ≤ 16,384
  • 1 ≤ N ≤ 4,096 and N is a power of two
  • All values are 32-bit floating point numbers
  • Absolute error ≤ 1e-4 and relative error ≤ 1e-4
  • Performance is measured with B = 8,192, N = 4,096
