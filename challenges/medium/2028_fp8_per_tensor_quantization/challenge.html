Implement per-tensor float32 to FP8 E4M3 quantization. Given a float32 array \(x\) of length \(N\), compute the per-tensor scale as \(\text{max}(|x|) / 448\) and quantize each element by dividing by the scale and converting to the OCP FP8 E4M3 format. This is the simplest FP8 quantization scheme — one scale factor for the entire tensor — used as the building block for more advanced per-token, per-channel, and block-wise FP8 quantization on NVIDIA H100 and AMD MI300X GPUs.

FP8 E4M3 Format

  • Bit layout: \([S|E_3|E_2|E_1|E_0|M_2|M_1|M_0]\), bias = 7, max normal = 448
  • Normal: \((-1)^S \times 2^{E-7} \times 1.M\)
  • Subnormal (exp=0000): \((-1)^S \times 2^{-6} \times 0.M\)
  • NaN: encode as 0x7F. Overflow: clamp to ±448

Algorithm

max_abs = max(|x[i]|) over all i in [0, N)
scale = max_abs / 448.0
If max_abs == 0: scale = 1.0
For each i in [0, N):
  y[i] = fp8_e4m3(x[i] / scale)
scale_out[0] = scale

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • FP8 encodings in y (uint8); scale in scale_out (float32, single-element tensor)
  • All computation must occur on the GPU

Example

Input:  x = [1.0, -2.0, 0.5, -0.25]
        N = 4

max(|x|) = 2.0
scale = 2.0 / 448 = 0.0044642857...
scaled = [224.0, -448.0, 112.0, -56.0]

Output: y = [0x76, 0xFE, 0x6E, 0xE6]
        scale_out = [0.0044642857...]
(224.0 → 0x76, -448.0 clamped → 0xFE,
 112.0 → 0x6E, -56.0 → 0xE6)

Constraints

  • 1 ≤ N ≤ 100,000,000
  • Input x is float32; output y is uint8; scale_out is float32
  • Performance is measured with N = 25,000,000
