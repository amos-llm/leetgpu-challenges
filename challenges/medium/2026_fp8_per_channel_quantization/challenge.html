Implement per-channel FP8 E4M3 quantization of a float32 weight matrix. This parallels INT8 per-channel quantization used in TensorRT and ONNX Runtime, replacing the INT8 target format with the OCP FP8 E4M3 format used on NVIDIA H100 and AMD MI300X GPUs. Given a float32 matrix \(x\) of shape \(M \times K\), compute per-output-channel scales and encode each element to FP8 E4M3.

Each column (channel) \(k\) gets an independent scale computed from the maximum absolute value within that column. The scale maps the column's range to the FP8 E4M3 representable range of ±448.

FP8 E4M3 Format

  • Bit layout: \([S|E_3|E_2|E_1|E_0|M_2|M_1|M_0]\), bias = 7, max normal = 448
  • Normal: \((-1)^S \times 2^{E-7} \times 1.M\)
  • Subnormal (exp=0000): \((-1)^S \times 2^{-6} \times 0.M\)
  • NaN: encode as 0x7F. Overflow: clamp to ±448

Algorithm

For each column k in [0, K):
  max_abs_k = max(|x[i,k]|) over all i in [0, M)
  scale_k = max_abs_k / 448.0
  If max_abs_k == 0: scale_k = 1.0
  For each i in [0, M):
    y[i,k] = fp8_e4m3(x[i,k] / scale_k)
  scales[k] = scale_k

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • FP8 encodings in y (uint8); per-channel scales in scales (float32)
  • All computation must occur on the GPU
  • Channels (columns) are independent and may be computed in parallel

Example

Input: Matrix \(x\) (\(M=2, K=3\)): \[ \begin{bmatrix} 120.0 & 0.5 & 0.0 \\ -60.0 & 1.0 & 0.0 \end{bmatrix} \]

Column 0: max_abs=120.0, scale=0.2678...
\(y[0,0] = \text{fp8_e4m3}(448.0) = \text{0x7E}\)
\(y[1,0] = \text{fp8_e4m3}(-224.0) = \text{0xF6}\)

Column 1: max_abs=1.0, scale=0.002232...
\(y[0,1] = \text{fp8_e4m3}(224.0) = \text{0x76}\)
\(y[1,1] = \text{fp8_e4m3}(448.0) = \text{0x7E}\)

Column 2: max_abs=0.0, scale=1.0
\(y[0,2] = \text{fp8_e4m3}(0.0) = \text{0x00}\)
\(y[1,2] = \text{fp8_e4m3}(0.0) = \text{0x00}\)

Output: Matrix \(y\): \[ \begin{bmatrix} \text{0x7E} & \text{0x76} & \text{0x00} \\ \text{0xF6} & \text{0x7E} & \text{0x00} \end{bmatrix} \]
scales = \([0.2678..., 0.002232..., 1.0]\)

Constraints

  • 1 ≤ M ≤ 65,536
  • 1 ≤ K ≤ 65,536
  • Input x is float32; output y is uint8; scales is float32
  • All tensors stored in row-major order
  • Performance is measured with M = 4,096, K = 4,096
