Implement MXFP4 (Microscaling FP4) activation quantization — the online step that runs at every inference forward pass, converting float32 activations into packed FP4 + E8M0 scales before feeding into hardware-accelerated MXFP4 matrix multiplication on AMD MI300X and NVIDIA Blackwell GPUs. MXFP4 is the OCP microscaling standard: a float32 activation matrix x[M, K] is quantized per-token (per-row), where M is the token count and K is the feature (contraction) dimension of the GEMM. Each row is divided into contiguous 1D blocks of 32 elements along K, each block shares one FP8 E8M0 scale factor (power-of-2 only), and each element is encoded to FP4 E2M1 (4-bit floating point). Per-token granularity is essential because outlier tokens (e.g. delimiter tokens, early tokens in attention) can have activations 10–100× larger than typical tokens; a single per-tensor scale would either clip these outliers or quantize most values to zero. Blackwell and MI300X tensor cores natively consume packed FP4 data with E8M0 scales, performing fused dequantization and matrix multiply-accumulate in a single hardware instruction — a 4× memory bandwidth reduction over BF16/FP16 at the cost of this quantization pass.

Formats

  • FP4 E2M1: 1 sign, 2 exponent (bias=1), 1 mantissa. 16 values: \(\{0, \pm 0.5, \pm 1, \pm 1.5, \pm 2, \pm 3, \pm 4, \pm 6\}\)
  • FP8 E8M0 scale: 8-bit unsigned exponent. Value = \(2^{e-127}\). Range from \(2^{-126}\) to \(2^{127}\). One scale per block of 32, adding only 0.25 bits/element overhead.
  • Packing: two 4-bit FP4 values packed per uint8 along the K dimension. High nibble = even index, low nibble = odd index. Consecutive elements along K are contiguous in memory, enabling coalesced GPU access.

FP4 E2M1 Encoding Table

0x0= 0.0   0x1= 0.5   0x2= 1.0   0x3= 1.5
0x4= 2.0   0x5= 3.0   0x6= 4.0   0x7= 6.0
0x8=-0.0   0x9=-0.5   0xA=-1.0   0xB=-1.5
0xC=-2.0   0xD=-3.0   0xE=-4.0   0xF=-6.0

Algorithm

For each token m in [0, M):
  For each block b in [0, K/32):
    max_abs = max(|x[m, b*32 + i]|) for i in [0, 32)
    scale = smallest_power_of_2_ge(max_abs / 6.0)
    E8M0 = log2(scale) + 127, clamped to [1, 254]; E8M0 = 0 if max_abs = 0
    scales[m, b] = E8M0
    For each i in [0, 32):
      nibble = nearest_FP4(x[m, b*32 + i] / float(scale))
    Pack nibbles[0:16] (high) and nibbles[16:32] (low) into y[m, b*16 : (b+1)*16]

Implementation Requirements

  • External libraries are not permitted
  • The solve function signature must remain unchanged
  • FP4 nibbles packed in y; E8M0 scales in scales
  • All computation must occur on the GPU
  • Each (token, block) pair is independent and may be computed in parallel

Example

Input: One token with \(K=32\) features: 

[-6.0, -4.0, -2.0, -0.5,  0.0,  0.5,  2.0,  4.0,
 -3.0, -1.0,  1.0,  3.0,  1.5, -1.5,  0.0,  6.0,
  0.0,  3.0, -1.5,  0.5,  2.0, -0.5, -3.0,  1.0,
  6.0,  4.0, -2.0,  0.0,  1.5, -4.0, -6.0, -1.0]

max_abs = 6.0, scale = \(2^0 = 1.0\), E8M0 = 127
\(-6.0/1.0 = -6.0 \to 0xF\), \(-4.0 \to 0xE\), \(-2.0 \to 0xC\), \(-0.5 \to 0x9\)
\(0.0 \to 0x0\), \(0.5 \to 0x1\), \(2.0 \to 0x4\), \(4.0 \to 0x6\), etc.

Output: \(y\) (\(M=1, K/2=16\)) = packed FP4 nibbles
scales (\(M=1, K/32=1\)) = [127]

Constraints

  • K is a multiple of 32
  • 1 ≤ M ≤ 8,192
  • 32 ≤ K ≤ 65,536
  • Input x is float32; output y and scales are uint8
  • The matrix x is stored in row-major order
  • Performance is measured with M = 4,096, K = 4,096
