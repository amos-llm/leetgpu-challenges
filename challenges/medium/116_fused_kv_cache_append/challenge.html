Implement a fused GPU kernel that appends a new key–value pair to the KV-cache and immediately computes decode-phase scaled dot-product attention using the updated cache. In autoregressive LLM inference, this is the critical path executed once per generated token. A naive implementation writes K_new and V_new to global memory, then reads the entire cache back for attention. Fusing the append with attention keeps the newly written data in registers and eliminates the round-trip through global memory.

The kernel must:

\[ \text{Write: } K_{\text{cache}}[\text{seq\_len}] \leftarrow K_{\text{new}}, \quad V_{\text{cache}}[\text{seq\_len}] \leftarrow V_{\text{new}} \] \[ \text{Attend: } \text{out}[h] = \sum_{i=0}^{\text{seq\_len}} \text{softmax}\!\left(\frac{Q[h] \cdot K_{\text{cache}}[0:\text{seq\_len}+1, h]^T}{\sqrt{D}}\right)_{\!i} \cdot V_{\text{cache}}[i, h] \]

The key optimization: when computing the attention score for position seq_len, use the register-resident K_new and V_new directly instead of reading them back from the cache. The cache must still be updated for future decode steps.

Implementation Requirements

  • Do not change the function signature.
  • Do not use external libraries beyond what is available in the starter.
  • K_cache and V_cache must be updated at position seq_len.
  • The kernel must fuse the cache append with the attention computation in a single kernel launch.
  • Attention is computed over positions 0 to seq_len (inclusive) after the append.

Example

H = 1, D = 2, seq_len = 1 (one existing token, append a second):

\[ Q = \begin{bmatrix} 1.0 & 0.0 \end{bmatrix} \quad K_{\text{new}} = \begin{bmatrix} 1.0 & 0.0 \end{bmatrix} \quad V_{\text{new}} = \begin{bmatrix} 3.0 & 4.0 \end{bmatrix} \] \[ K_{\text{cache}}[0] = \begin{bmatrix} 1.0 & 1.0 \end{bmatrix} \quad V_{\text{cache}}[0] = \begin{bmatrix} 2.0 & 3.0 \end{bmatrix} \]

After appending K_new at seq_len=1, the attention over K_cache[:2] and V_cache[:2] produces:

\[ \text{out} \approx \begin{bmatrix} 2.5000 & 3.5000 \end{bmatrix} \]

Constraints

  • 1 ≤ H ≤ 64 (number of attention heads)
  • 2 ≤ D ≤ 256 (head dimension)
  • 0 ≤ seq_len < cache capacity (i.e. K_cache.shape[0])
  • Q, K_new, V_new, K_cache, V_cache, and output are float32
  • Performance is measured with H = 32, D = 128, seq_len = 2,047
